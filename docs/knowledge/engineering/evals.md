# LLM 评测方法

## 自动评测

### 1. Perplexity
- 衡量模型对文本的预测能力
- 越低越好

### 2. BLEU / ROUGE
- 文本相似度
- 适用于有标准答案的任务

### 3. 基准测试

| 基准 | 测试能力 |
|------|----------|
| MMLU | 多学科知识 |
| HumanEval | 代码生成 |
| GSM8K | 数学推理 |
| TruthfulQA | 事实准确性 |

## 人工评测

### 维度
- 有用性（Helpfulness）
- 准确性（Accuracy）
- 安全性（Safety）
- 流畅性（Fluency）

### 方法
- A/B 测试
- Elo 评分
- 偏好排序

## Agent 专用评测

| 指标 | 描述 |
|------|------|
| 任务完成率 | 是否完成目标 |
| 步骤效率 | 用了多少步 |
| 工具调用准确率 | 是否正确使用工具 |
| 幻觉率 | 是否产生错误信息 |

## 面试要点

1. **自动评测的局限性？**
   - 无法完全替代人工判断

2. **如何评测 Agent 效果？**
   - 端到端任务完成度 + 中间步骤质量
