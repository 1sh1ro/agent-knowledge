# Tokenizer（分词器）

## 基本概念

Tokenizer 将文本转换为模型可以处理的 token 序列。

## 主要类型

| 类型 | 原理 | 代表 | 特点 |
|------|------|------|------|
| BPE | 合并高频字符对 | GPT-2/3/4 | 平衡词表大小和覆盖率 |
| WordPiece | 基于概率合并 | BERT | 处理未登录词 |
| SentencePiece | 无语言依赖 | T5, LLaMA | 直接处理原始文本 |
| Unigram | 基于概率剪枝 | XLNet | 更灵活 |

## BPE 算法流程

1. 初始化词表为所有字符
2. 统计所有字符对出现频率
3. 合并最高频的字符对
4. 重复直到达到目标词表大小

## 面试要点

1. **为什么需要 Tokenizer？**
   - 将文本离散化，映射为向量

2. **不同 Tokenizer 的区别？**
   - BPE 贪婪合并，WordPiece 基于概率

3. **中文如何处理？**
   - 通常按字或子词切分
