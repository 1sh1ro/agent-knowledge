# RLHF（人类反馈强化学习）

## 训练流程

```
预训练模型
    ↓
SFT（监督微调）
    ↓
奖励模型训练
    ↓
PPO 强化学习
```

## 1. SFT（Supervised Fine-Tuning）

- 使用高质量指令数据微调
- 让模型学会遵循指令格式

## 2. 奖励模型（Reward Model）

- 输入：Prompt + 回答
- 输出：评分（越高越好）
- 训练数据：人工标注的偏好对

## 3. PPO（Proximal Policy Optimization）

- 策略：当前语言模型
- 奖励：奖励模型评分 + KL 散度约束
- 目标：最大化奖励，同时不偏离 SFT 模型太远

## 替代方案

| 方法 | 特点 |
|------|------|
| DPO | 直接偏好优化，无需奖励模型 |
| KTO | 二元反馈优化 |
| ORPO |  odds ratio 偏好优化 |

## 面试要点

1. **RLHF 为什么有效？**
   - 将人类偏好编码到奖励函数中

2. **PPO 中的 KL 散度约束作用？**
   - 防止模型崩溃，保持生成多样性
