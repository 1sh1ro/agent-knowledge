### 高频问题与参考答案

#### 1. ReAct 循环里消息格式怎么设计的？tool_response 用 user 还是 assistant 角色传回？为什么？

**正确答案：tool_response 要用 user 角色传回。**

原因：
- tool_response 是外部系统返回的内容，不是模型生成的。
- 如果放 assistant 角色，模型会以为自己说了这些话，后续推理就会混乱。
- 放 user 角色，明确标识这是"环境观察（Observation）"，模型据此进行下一轮 Thought。

消息格式示例：
```
system:    你是一个 ReAct Agent...
user:      用户的原始问题
assistant: Thought: 我需要搜索...
           Action: search_tool(query="...")
user:      Observation: 搜索结果返回...（tool_response，外部系统返回）
assistant: Thought: 根据搜索结果...
           Action: final_answer(...)
```

#### 2. 训练流程：Agentic CPT → SFT → RL 三阶段，为什么 SFT 时要 mask 掉 observation tokens？

**答案：因为 observation 是外部环境返回的结果，不是模型生成的。**

- 如果对 observation 也算 loss，模型会试图学习预测搜索结果，这件事既不可能也没有意义。
- SFT 时只对模型应该生成的部分（Thought / Action）计算 loss，observation 部分 mask 掉。
- 这保证模型学习的是"如何推理和决策"，而不是"如何预测环境返回"。

#### 3. 你的 Agent 调了三个工具就死循环了，异常处理写在哪？

**答案要能说出来三层：**

1. **最大迭代次数硬限制**：设置 Agent 循环的最大步数（如 10 步），超过直接终止并报错。
2. **循环模式检测**：检测 Agent 是否在重复相同的工具调用（如连续 3 次调用相同工具相同参数），检测到则中断。
3. **异常回退策略**：重试 → 降级（换更简单策略/规则引擎兜底）→ 人工介入。

> 这道题考的不是背概念，是你有没有真正跑过生产级的 Agent。
