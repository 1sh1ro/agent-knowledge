## Q1: 解释 Agent = Model + Harness 公式

面试官在白板上写下一行字：

> **Agent = Model + Harness**

然后转过身："讲讲。"

### 参考回答

Model 是推理引擎。你给它一个问题，它给你一个回答。

Harness 是剩下的一切，包含7个组件：

| 组件 | 作用 | 类型 |
|------|------|------|
| **CLAUDE.md** | 告诉 Agent 项目规矩 | 静态 |
| **Hooks** | 做事时的动态检查 | 事件驱动 |
| **Skills** | 可复用的工作流封装 | 跨项目 |
| **Subagents** | 独立上下文的子 Agent | 并行/隔离 |
| **MCP** | 工具协议标准 | 外部能力接入 |
| **Trace** | 决策日志 | 可观测性 |
| **Rollback** | 状态回滚 | 容错 |

### 记忆口诀

> **Model 管思考，Harness 管运行。你以为在训练模型，其实在设计环境。**

核心洞察：prompt 只解决"你对 Agent 说什么"。Harness 解决的是："Agent 怎么运行、用什么工具、出错怎么办、结果怎么验证"。

## Q2: 四次范式跃迁

面试官在白板上画了一条时间线，标注了四个节点：

> "Prompt Engineering 过时了，Context Engineering 也快过时了。这四次跃迁怎么理解？"

### 四次范式跃迁详解

| 范式 | 解决什么 | 留下的问题 |
|------|---------|-----------|
| **Prompt Engineering** | 怎么向模型表达意图 | 意图能表达复杂逻辑吗？ |
| **Context Engineering** | 怎么给模型提供正确信息 | 信息爆炸了怎么办？信息过时了怎么办？ |
| **Harness Engineering** | 怎么设计和管控 Agent 的全部运行环境 | 环境怎么设计？循环怎么控制？ |
| **Loop Engineering** | 怎么设计 Agent 自身的执行循环 | 多步、异步、长时任务的编排 |

### 核心洞察

每一次跃迁，都是在解决上一代留下的瓶颈：

- **Prompt** 是单点输入，是一次性的
- **Context** 升级到信息架构，但很快发现：信息再多，也需要一套机制来管理权限、工具、状态、回滚
- 这就是 **Harness**。但 Harness 还缺一个东西——循环本身的设计
- 所以演进到 **Loop Engineering**

---

## 面经总结

京东三面的核心考法不是让你背定义，而是让你**在白板上解构 Agent**：

1. 能否把 Agent 的组成拆成 Model + Harness 两大类？
2. 能否解释 Harness 的7个组件各起什么作用？
3. 能否理解四次范式跃迁的演进逻辑？
4. 能否把"你以为在训练模型，其实在设计环境"这句话说清楚？

面试官想听的，不是标准答案，而是你对 Agent 体系的理解深度——从单点到体系，从静态到动态，从输入到循环。
