### 参考答题要点

**Q3 完整 RAG 系统核心部分**：
- 文档解析与预处理（上传/分块）
- Embedding 模型（文本向量化）
- 向量数据库（存储与索引，如 Milvus/Chroma）
- 检索策略（相似度检索 + 可选 Rerank）
- 上下文组装与 Prompt 工程
- 生成与流式输出（SSE）
- 效果评估（召回率/精度）

**Q23 MCP / Function Call / A2A 区别**：
- **Function Call**：模型层面的能力，LLM 根据用户意图生成结构化工具调用参数（JSON），由开发者侧执行。
- **MCP（Model Context Protocol）**：协议层标准，定义 AI 助手与外部数据源/工具的连接规范，类似"AI 的 USB 接口"，解决工具接入的标准化问题。
- **A2A（Agent-to-Agent）**：Agent 间通信协议，解决多 Agent 协作时的任务委派、状态同步、消息传递问题。
- 三者层级：Function Call（模型能力）→ MCP（工具连接协议）→ A2A（Agent 间协作协议）。

**Q27 Claude Code 实现原理要点**：
- 基于 LLM 的 Agentic Coding 工具，核心是 ReAct 循环 + 工具调用（文件读写、命令执行、搜索等）。
- 通过 System Prompt 约束行为，将代码库作为上下文注入。
- 工具调用采用 Function Calling，结果以 user 角色回传（外部系统返回，非模型生成）。
- 具备 Plan 模式（先规划后执行）与长短期记忆管理。

### 多模态 RAG 图片向量化参考

图片在 RAG 系统中的向量化检索方案：
1. **特征提取**：使用 CLIP / BLIP 等多模态模型，将图片编码为高维向量。
2. **跨模态对齐**：CLIP 通过对比学习让图文向量在同一空间对齐，实现"用文本检索图片"。
3. **存储索引**：图片向量存入向量数据库（Milvus/Weaviate），建立 ANN 索引。
4. **检索**：用户文本 query → 文本编码 → 与图片向量做余弦相似度检索 → 返回 Top-K 图片。
5. **难点**：细粒度对齐（局部区域）、跨场景误召回、多模态融合权重。

### Skill 分层体系设计参考

- **静态 Skill**：预定义的高频能力（如文件操作、代码搜索、命令执行），随系统启动加载。
- **动态 Skill**：运行时按需加载的能力，可由用户构造或 Agent 自动沉淀。
- **匹配机制**：用户输入 → 意图识别 → Skill 向量检索 → Top-K 候选 → LLM 选择最优 Skill。
- **沉淀机制**：Agent 执行成功的操作序列可被封装为新 Skill，存入 Skill 库供后续复用。

### 长短期记忆设计参考

- **短期记忆**：当前会话的对话历史与任务状态，存于 Context Window。
- **静态长期记忆**：用户偏好、固定知识、项目约定，启动时加载。
- **动态长期记忆**：运行时积累的经验与事实，Embedding 后存入向量库。
- **触发存储**：关键事实提取 + 重要性评分阈值，避免过快积累。
- **召回决策**：当前任务向量化 → 检索相关长期记忆 → 相关性分数过滤 → 注入上下文。
- **压缩机制**：摘要缓冲（Summary Buffer），早期内容摘要化，控制总 token 在模型窗口 60-70% 以内。

   - 格式失败：未按约定 JSON/结构化格式返回
   - 内容失败：输出"无法回答"或无关内容，甚至幻觉编造

2. **如何处理模型输出失败？工业界兜底方案？**
