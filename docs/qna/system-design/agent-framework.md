# 设计 Agent 框架

## 核心模块

```
┌─────────────────────────────────────┐
│           Agent 框架                 │
├─────────────────────────────────────┤
│  Planner（规划器）                   │
│  - 任务分解                          │
│  - 步骤生成                          │
├─────────────────────────────────────┤
│  Memory（记忆）                      │
│  - 短期记忆（上下文）                 │
│  - 长期记忆（向量库）                 │
├─────────────────────────────────────┤
│  Tool Manager（工具管理）            │
│  - 工具注册                          │
│  - 工具选择                          │
│  - 参数生成                          │
├─────────────────────────────────────┤
│  Executor（执行器）                  │
│  - 调用工具                          │
│  - 处理结果                          │
└─────────────────────────────────────┘
```

## 关键设计

### 1. 规划策略
- ReAct: Thought → Action → Observation
- Plan-and-Solve: 先计划后执行
- Reflection: 执行后反思

### 2. 工具调用
```json
{
  "tool": "search",
  "parameters": {
    "query": "..."
  }
}
```

### 3. 错误处理
- 工具调用失败 → 重试或换工具
- 结果异常 → 重新规划

## 面试要点

1. **如何设计可扩展的工具系统？**
   - 插件化、标准化接口

2. **Agent 和 Workflow 的区别？**
   - Agent 自主决策，Workflow 预定义流程
