# 设计推荐系统

## 架构

```
用户行为 → 特征工程 → 召回 → 排序 → 重排 → 展示
```

## 召回层

| 方法 | 原理 |
|------|------|
| 协同过滤 | 用户/物品相似度 |
| 向量召回 | Embedding 相似度 |
| 热门推荐 | 全局热度 |
| 规则召回 | 业务规则 |

## 排序层

- 特征：用户、物品、上下文
- 模型：LR、GBDT、DeepFM

## 冷启动

| 场景 | 策略 |
|------|------|
| 新用户 | 热门推荐、问卷引导 |
| 新物品 | 内容相似、探索机制 |

## 面试要点

1. **召回和排序的区别？**
   - 召回：粗筛，保证相关性
   - 排序：精排，优化目标

2. **如何解决冷启动？**
   - 内容信息、探索机制、迁移学习
