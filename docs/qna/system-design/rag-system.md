# 设计 RAG 系统

## 架构图

```
用户查询
  ↓
Query 理解（改写、扩展）
  ↓
检索模块（向量搜索 + 关键词搜索）
  ↓
重排序（Rerank）
  ↓
上下文拼接
  ↓
LLM 生成
  ↓
后处理（格式化、验证）
```

## 核心组件

### 1. 文档处理
- 分块（Chunking）
- 清洗（去重、去噪）
- 向量化（Embedding）

### 2. 检索
- 向量检索（语义匹配）
- 关键词检索（精确匹配）
- 混合检索（Hybrid Search）

### 3. 生成
- Prompt 工程
- 上下文窗口管理
- 引用标注

## 优化方向

| 方向 | 方法 |
|------|------|
| 召回率 | Query 扩展、多路召回 |
| 准确率 | Rerank、过滤 |
| 延迟 | 缓存、索引优化 |
| 幻觉 | 引用验证、置信度 |

## 面试要点

1. **Chunk size 如何选择？**
   - 太小：信息不完整
   - 太大：噪声多、检索不精确

2. **如何解决幻觉？**
   - 引用原文、置信度阈值、后验证
