# 行为面（Behavioral Questions）

## STAR 法则

- **S**ituation: 描述背景和情境
- **T**ask: 说明你的任务和目标
- **A**ction: 你采取了什么行动
- **R**esult: 最终结果如何

## 高频问题

| 问题 | 考察点 |
|------|--------|
| 介绍最有挑战的项目 | 项目能力、技术深度 |
| 遇到最难的技术难题 | 问题解决能力 |
| 为什么想来我们公司 | 动机与规划 |
| 最近在学什么 | 学习能力 |
| 和团队成员的冲突 | 沟通能力 |

## 回答模板

### 项目介绍

```markdown
**项目背景**: 为了解决 XXX 问题，公司决定...

**我的角色**: 作为 XXX，负责 XXX 模块

**技术方案**: 
- 采用了 XXX 架构
- 使用 XXX 技术解决 XXX 问题

**核心挑战**: 
- 遇到了 XXX 困难
- 通过 XXX 方法解决

**最终效果**: 
- 实现了 XXX 提升
- 获得了 XXX 成果
```

## 准备建议

1. 准备 3-5 个项目经历
2. 每个项目准备不同角度的故事
3. 量化结果（数据说话）
4. 体现个人贡献
