## 二、2026年 Agent 面试趋势分析

### 2.1 面试形式变化

根据多个面经分析，2026年 AI Agent 面试呈现以下趋势：

1. **告别"八股文时代"**
   - 不再问"什么是自注意力机制"等基础概念题
   - 面试官更关注实际项目经验和问题解决能力

2. **直接考察 Agent 框架使用**
   - 不再问"你怎么看AI"
   - 直接甩场景，让你当场出方案
   - 重点考察：用哪个 Agent 框架、为什么选它

3. **工程实战成为重点**
   - 三轮技术面全是工程实战
   - 不考算法，只考工程能力（如科大讯飞面经）

### 2.2 高频考察方向

| 排名 | 考察方向 | 占比 | 说明 |
|-----|---------|-----|------|
| 1 | Agent 项目拷打 | 30% | 项目经历、架构设计、问题解决 |
| 2 | RAG 相关 | 25% | Chunking策略、召回优化、幻觉控制 |
| 3 | Prompt Engineering | 20% | Prompt优化、Few-shot、CoT |
| 4 | LangChain 应用 | 15% | 工具调用、Memory管理 |
| 5 | LLM 基础 | 10% | Attention机制、位置编码等 |

## 四、面试核心问题分类

### 4.1 项目经历类（必问）

```markdown
1. 请介绍一下你做过的Agent项目？
2. 项目的整体架构是怎样的？
3. 遇到过最大的技术挑战是什么？如何解决的？
4. 项目的效果如何评估？
5. 为什么选择这个技术方案？有什么替代方案？
```

### 4.2 RAG 相关（高频）

```markdown
1. RAG的流程是什么？有哪些优化点？
2. 文本分块策略有哪些？如何选择chunk size？
3. 如何提升召回率？
4. 如何解决RAG的幻觉问题？
5. 向量数据库有哪些？如何选型？
6. Hybrid Search（混合检索）了解吗？
```

### 4.3 LangChain/Agent框架

```markdown
1. 你用过哪些Agent框架？有什么区别？
2. LangChain的核心组件有哪些？
3. 工具调用（Tool Calling）是如何实现的？
4. Memory是如何管理的？
5. Agent的规划能力是如何实现的？
```

### 4.4 Prompt Engineering

```markdown
1. 如何写一个好的Prompt？
2. Few-shot和Zero-shot的区别？
3. Chain-of-Thought是什么？如何应用？
4. Prompt优化的常见技巧有哪些？
```

### 4.5 LLM 基础

```markdown
1. Transformer的结构是怎样的？
2. Attention的计算过程？
3. 位置编码有哪些？RoPE的原理？
4. GPT和BERT的区别？
5. 模型的推理加速方法？
```

## 六、面试准备建议

### 6.1 知识储备

| 类别 | 推荐学习内容 | 资源推荐 |
|-----|-------------|---------|
| LLM基础 | Transformer、Attention、位置编码 | 本项目 knowledge/llm-basics |
| Agent框架 | LangChain、LlamaIndex、AutoGPT | 本项目 knowledge/agent-architecture |
| RAG系统 | Chunking、召回、向量数据库 | 本项目 knowledge/engineering/rag.md |
| Prompt工程 | Few-shot、CoT、Prompt优化 | 本项目 qna/ |

### 6.2 项目准备

1. **准备1-2个核心项目**
   - 熟悉项目架构和细节
   - 能说清楚技术选型原因
   - 能应对深挖和追问

2. **准备项目亮点**
   - 如何优化性能
   - 如何解决实际问题
   - 取得了什么效果

### 6.3 面试技巧

1. **STAR法则**：Situation-Task-Action-Result
2. **深挖项目**：每个项目准备5-10个深入问题
3. **技术广度**：了解多种技术方案及其优劣
4. **问题驱动**：能提出问题并给出解决方案
